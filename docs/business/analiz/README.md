# Ad-hoc Analizler

Tarihli, tek-konulu, hızlı analiz notları. Pipeline / KARAR-LOG'a dönüşmeden önceki **düşünme alanı**.

## Konvansiyon

- **İsimlendirme:** `YYYY-MM-DD-konu.md`
- Örnek: `2026-04-21-koc-oto-girisimleri.md`, `2026-05-02-sigorta-kasko-akisi.md`
- Her analiz bağımsız okunabilir olmalı — başında 2-3 cümle bağlam + altta "sonuç / öneri" bölümü.

## Yaşam döngüsü

1. Analiz burada yazılır (düşünme).
2. Olgunlaştığında → ilgili kategori klasörüne kalıcı doküman çıkar (örn. `ortakliklar/sigorta-ortakligi.md`).
3. Karar netleşirse → [../KARAR-LOG.md](../KARAR-LOG.md) girişi.
4. Orijinal analiz burada kalır; artık **arşiv** niteliğinde — silinmez ki kararın gerekçesi geri izlenebilsin.

## İlke

Analiz yazarken **cesur ol** — olgunlaşmamış fikir buraya yakışır. Kalıcı dokümanda belirsiz kal, analizde iddialı ol.
